# opensource-webflux
"실업무"에서 오픈소스로 이용할 수 있도록, 서비스성향/기술/트렌드를 고려해 만든 웹플럭스 입니다.


# 0. 목차

### 1.프로젝트 설계

1-1. 컨셉 시안 ( 목표,뜻 ) </br>
option1. 서비스 "시장" </br>
option2. 서비스 "유저" 시나리오 ( user uml로 보는 액션과 데이터 ) </br>
option3. 신규 고객 "경험" </br>

1-2.시스템 R&R ( 액션 ) </br>
option1. 컨셉 고려 </br>
option2. 주체들 고려 ( 유저 ⊃ 고객 ⊃ PM ⊃ 개발자 ⊃ 나 / 바깥-> 안쪽 사람들 고려 ) </br>
컨센에 맞는 주체 범위 선정 </br>
범위에 따른 기술 선정, 트렌드 기술 선정 </br>

1-2-1. 시스템 구성도 </br>
1-2-2. 프로젝트 구성도 </br>
option1. 프로젝트 user flow </br>
